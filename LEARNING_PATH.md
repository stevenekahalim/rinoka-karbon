## Important Principles

### 1. **Build, Don't Just Watch**
- Watching 10 tutorials < Building 1 project
- Type every line yourself, don't copy-paste
- Break things and fix them

### 2. **Start Simple, Add Complexity**
- Make it work (ugly is fine)
- Make it clean (refactor)
- Make it fast (optimize)

### 3. **Google is Your Friend**
- "How to [thing] in JavaScript"
- Read StackOverflow answers
- Check documentation

### 4. **Daily Consistency > Weekend Marathons**
- 1 hour daily > 7 hours on Sunday
- Build momentum
- Your brain needs time to process

### 5. **Document Your Journey**
- Keep notes on what you learn
- Comment your code
- Build a portfolio

## When You Get Stuck

### The 30-Minute Rule:
1. **Try to solve it** (15 min)
   - Read error message
   - Google the error
   - Check documentation

2. **Ask for help** (if still stuck)
   - Explain what you tried
   - Show your code
   - Describe expected vs actual result

3. **Learn from solution**
   - Understand WHY it works
   - Try to break it
   - Apply to different problem
